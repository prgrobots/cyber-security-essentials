Hashing Algorithms

Hashing is a tool that ensures data integrity by taking binary data (the message) and producing a fixed-length representation called the hash value or message digest. Integrity is demonstrated if the file hash is unchanged after transport, after a period of storage, or tested before processing.
Hashing is a one-way mathematical function that is relatively easy to compute, but significantly harder to reverse. Grinding coffee beans is a good analogy of a one-way function. It is easy to grind coffee beans, but it is almost impossible to put all of the tiny pieces back together to rebuild the original beans.

Hashing Algorithms

Signatures and the Law

Digital signatures provide the same functionality as handwritten signatures for electronic documents.
A digital signature is used to determine 	if someone has edited a document after the user signs it.

In many countries, digital signatures 	have the same legal importance as a 	manually signed document.

Database Integrity

Databases provide an efficient way to store, retrieve, and analyze data.
As data collection increases and data becomes more sensitive, it is important for cybersecurity professionals to protect the growing number of databases.
Data integrity refers to the accuracy, consistency, and reliability of data stored in a database.

## Slide 17

OFFICIAL

Database Integrity

The four database integrity rules or constraints are as follows:
Entity Integrity: All rows must have a unique identifier called a Primary Key.
Domain Integrity: All data stored in a column must follow the same format and definition.
Referential Integrity: Table relationships must remain consistent. Therefore, a user cannot delete a record which has other records related to first one. So, a customer record can’t be deleted before deleting all related (1:Many) transaction records belonging to that customer.

User-defined Integrity: A set of rules defined by a user which does not belong to one of the other categories. For example, a customer places a new order. The user first checks to see if this is a new customer. This is usually done by first checking if there is an existing customer record with the same name, address or phone number or Date of Birth (DoB).If no existing customer record exists, then the user adds the new customer to the customers table.

## Slide 19

OFFICIAL

Database Validation

Data validation rules check that new inputted data falls within the parameters defined by the database designer. Data validation rules helps to ensure the completeness, accuracy and consistency of data. The criteria used in a validation rule include:
Size – checks the number and type of characters in a data field.
Format (input mask) – checks that the data conforms to a specified format
Consistency – checks for the consistency of codes in related data items
Range – checks that data lies within a min and maximum value
Check digit – provides for an extra calculation to generate a check digit for error detection.

## Slide 20

OFFICIAL

Database Integrity Requirements

Maintaining proper filing is critical in maintaining the trustworthiness and usefulness of the data within the database.
Tables, records, fields, and data within each field make up a database.
In order to maintain the integrity of the relational database tables (or Entities), database developers must follow entity normalization rules.
Entity integrity is an integrity rule, stating that every table must have a primary key and that the column or columns chosen to be the primary key must be unique and not NULL.
Null in a database signifies missing or unknown values.

Database Integrity Requirements

Another important integrity check is referential integrity which deals with foreign keys. A foreign key in “many” table references a primary key in a “one” table. The primary key for a table uniquely identifies entities (rows) in the table. Referential integrity maintains the integrity of foreign keys.

## Slide 22

Database Integrity Requirements

Domain integrity ensures all data items in a column fall within a defined set of valid values. Each column in a table has a defined set of values, such as the set of all numbers for credit card numbers, social security numbers, or email addresses.
Limiting values assigned to an instance of that column enforces domain integrity – which can be as simple as choosing the correct data type, length and or format for a column.
